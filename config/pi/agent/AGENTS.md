# Pi 全局规则

- 行动前读取距离当前工作目录最近的项目说明。
- 修改已跟踪文件前，先检查仓库状态。
- 保留无关改动和任务开始前已有的改动。
- 对于实现类请求，直接完成范围内的修改和验证；普通的执行步骤无需逐项询问。
- 仅当答案会实质影响结果时，才要求用户澄清。
- 当请求仅要求调查或诊断时，只分析和报告，不修改文件。
- 优先运行与改动范围直接相关的最小验证命令。
- 未经用户明确要求，不得提交、推送、部署、发布、安装依赖或修改系统配置。
- 不得暴露凭据，也不得将机器本地密钥加入仓库。
- 执行破坏性、不可逆或对外可见的操作前，必须先询问。
- 完成任务时，说明修改的文件、执行的验证以及仍然存在的风险。
- 先说明结果，保持回复简洁。
